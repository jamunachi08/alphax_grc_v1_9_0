# AlphaX GRC User Manual

## 1. Getting Started
- Open the **AlphaX GRC** module from Desk.
- Start with **AlphaX GRC Settings** and confirm review frequency, enabled modules, and focus frameworks.
- Create Frameworks, Controls, Risks, Policies, Vendors, Assessments, and Audit Plans in that order.

## 2. Daily User Flows
### Risk Owner
1. Open **GRC Risk Register**.
2. Create or update a risk.
3. Set impact, likelihood, treatment plan, and next review date.
4. Link related controls, incidents, findings, and remediation actions.

### Compliance Officer
1. Open **GRC Assessment** or **GRC Regulatory Obligation**.
2. Record the framework or regulation under review.
3. Capture evidence, maturity score, and compliance status.
4. Raise findings or remediation actions if gaps are found.

### Auditor
1. Create **GRC Audit Plan**.
2. Add **GRC Audit Workpapers** for testing steps and samples.
3. Record **GRC Audit Findings** and management responses.
4. Track **GRC Remediation Action** until verification and closure.

### Vendor Owner
1. Maintain **GRC Vendor** profile and criticality.
2. Run **GRC Vendor Assessment** with questionnaire items.
3. Track due diligence, open gaps, and next review date.

### Privacy Officer
1. Maintain **GRC Privacy Processing Activity**.
2. Track **GRC Data Subject Request** lifecycle.
3. Monitor **GRC Regulatory Obligation** records for PDPL.

## 3. Recommended Cadence
- Weekly: update incidents, critical risks, overdue remediation.
- Monthly: review high risks, findings, open assessments, vendor actions.
- Quarterly: policy reviews, access reviews, maturity scoring, board reporting.
- Annually: audit universe refresh, internal audit plan, policy republishing.

## 4. Bilingual Operation
- Use English or Arabic labels according to site language.
- Reports and print formats should support LTR and RTL layouts.
- Store Arabic titles where available for packs, policies, and framework records.
